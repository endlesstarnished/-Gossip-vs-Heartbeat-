=============================================================================
РАСЧЕТ ПРОПУСКНОЙ СПОСОБНОСТИ ПРОТОКОЛОВ ОБНАРУЖЕНИЯ ОТКАЗОВ
=============================================================================

ИСХОДНЫЕ ДАННЫЕ:
- Количество узлов (N) = 50
- Процент отказов = 10%
- Потеря пакетов = 5%
- Размер пакета (packet_size) = 100 байт (типовое значение)

=============================================================================
1. GOSSIP ПРОТОКОЛ
=============================================================================

Параметры:
- interval = 0.2 секунды
- fanout = 3 (каждый узел шлет слух 3 случайным соседям)

Формула пропускной способности:
Bandwidth (бит/с) = (N × fanout × packet_size × 8) / interval

Расчет:
1. Сообщений в секунду = (50 × 3) / 0.2 = 750 сообщений/с
2. Байт в секунду = 750 × 100 = 75 000 байт/с
3. Бит в секунду = 75 000 × 8 = 600 000 бит/с
4. Переводим в Мбит/с = 600 000 / 1 000 000 = 0.6 Мбит/с

ИТОГО для Gossip: 0.6 Мбит/с

=============================================================================
2. HEARTBEAT ПРОТОКОЛ
=============================================================================

Параметры:
- interval = 0.2 секунды
- timeout = 1.0 секунда

Формула пропускной способности:
Bandwidth (бит/с) = (N × packet_size × 8) / interval

Расчет:
1. Сообщений в секунду = 50 / 0.2 = 250 сообщений/с
2. Байт в секунду = 250 × 100 = 25 000 байт/с
3. Бит в секунду = 25 000 × 8 = 200 000 бит/с
4. Переводим в Мбит/с = 200 000 / 1 000 000 = 0.2 Мбит/с

ИТОГО для Heartbeat: 0.2 Мбит/с

=============================================================================
3. СРАВНЕНИЕ
=============================================================================

| Протокол    | Сообщений/с | Трафик (Мбит/с) |
|-------------|-------------|-----------------|
| Gossip      | 750         | 0.60            |
| Heartbeat   | 250         | 0.20            |

Вывод: Gossip создает в 3 раза большую нагрузку на сеть (0.6 Мбит/с против 0.2 Мбит/с)

=============================================================================
4. ДЛЯ СИСТЕМЫ ИЗ 100 УЗЛОВ
=============================================================================

Gossip (interval=0.3с, fanout=3):
- Сообщений/с = (100 × 3) / 0.3 = 1000 сообщений/с
- Трафик = 0.8 Мбит/с

Heartbeat (interval=0.5с, timeout=2.0с):
- Сообщений/с = 100 / 0.5 = 200 сообщений/с
- Трафик = 0.16 Мбит/с
